משימה — תיקון תבנית הצעת מחיר מסוג סיור

יש לעדכן את תבנית הצעת המחיר מסוג **סיור** כך שטבלת העלויות תהיה מבוססת רכיבים גמישים.

חשוב מאוד:
לא לשנות סטטוסים.
לא לשנות הרשאות.
לא לשנות חתימה.
לא לשנות אישורים.
לא לשנות Supabase / DB אם לא נדרש.
לא לשנות תבניות אחרות.
לא לבצע ריפקטורינג כללי.
לא לפגוע בהצעות קיימות ככל שניתן.

המטרה:
בחלון הצד של עריכת הצעת מחיר מסוג סיור, המשתמש יוכל להוסיף שורות לפי רכיב.
כל שורה תוצג בטבלת ההצעה עם:

* רכיב / תיאור
* כמות
* מחיר
* סה״כ

החישוב לכל שורה:

`כמות × מחיר = סה״כ`

החישוב הכללי:

סכום כל שורות ה־סה״כ.

מבנה הרכיבים:

| רכיב לבחירה בחלון העריכה | מה יוצג בטבלה                    | כמות         | מחיר         | סה״כ        |
| ------------------------ | -------------------------------- | ------------ | ------------ | ----------- |
| כיתה / קבוצה             | למילוי חופשי — שם הכיתה / הקבוצה | למילוי חופשי | למילוי חופשי | כמות × מחיר |
| מדריך                    | מדריך מלווה                      | למילוי חופשי | למילוי חופשי | כמות × מחיר |
| מיניבוס                  | הסעה-מיניבוס                     | למילוי חופשי | למילוי חופשי | כמות × מחיר |
| אוטובוס                  | הסעה-אוטובוס                     | למילוי חופשי | למילוי חופשי | כמות × מחיר |

הערה חשובה:
לא להפוך בין מיניבוס לאוטובוס.

ההיגיון הנכון הוא:

* מיניבוס מציג: `הסעה-מיניבוס`
* אוטובוס מציג: `הסעה-אוטובוס`

דרישות ממשק עריכה:

1. בחלון צד של עריכת הצעת סיור תהיה אפשרות להוסיף שורת עלות.
2. בכל שורה תהיה בחירה של רכיב:

   * כיתה / קבוצה
   * מדריך
   * מיניבוס
   * אוטובוס
3. אם הרכיב הוא כיתה / קבוצה:

   * לאפשר מילוי חופשי של שם הכיתה / הקבוצה שיוצג בטבלה.
4. אם הרכיב הוא מדריך:

   * להציג בטבלה אוטומטית: `מדריך מלווה`.
5. אם הרכיב הוא מיניבוס:

   * להציג בטבלה אוטומטית: `הסעה-מיניבוס`.
6. אם הרכיב הוא אוטובוס:

   * להציג בטבלה אוטומטית: `הסעה-אוטובוס`.
7. לכל רכיב יהיו שדות:

   * כמות
   * מחיר
   * סה״כ מחושב אוטומטית
8. לאפשר יותר משורה אחת מאותו רכיב אם צריך.

דרישות טבלת ההצעה / PDF:

בטבלת הצעת המחיר מסוג סיור יוצגו העמודות:

| רכיב | כמות | מחיר | סה״כ |

בסוף הטבלה תופיע שורת:

`סה״כ`

שהיא סכום כל שורות הסה״כ.

עיצוב ורוחב טבלה:

יש להקטין את הרוחב הכולל של הטבלה כך שתהיה קומפקטית ומסודרת.

רוחבי עמודות ויישור:

1. עמודת `רכיב`

   * רוחב: `125px`
   * כותרת מיושרת לימין
   * תוכן מיושר לימין

2. עמודת `כמות`

   * רוחב: `75px`
   * כותרת מיושרת לאמצע
   * תוכן מיושר לאמצע

3. עמודת `מחיר`

   * רוחב: `85px`
   * כותרת מיושרת לאמצע
   * תוכן מיושר לאמצע

4. עמודת `סה״כ`

   * רוחב: `85px`
   * כותרת מיושרת לאמצע
   * תוכן מיושר לאמצע

הרוחב הכולל המומלץ לטבלה:

`370px`

כלומר:
`125 + 75 + 85 + 85 = 370px`

אפשר להשתמש ב־`width: 370px` או `max-width: 370px`, לפי מה שמתאים לתצוגת המסמך, אבל לא להשאיר את הטבלה רחבה מדי.

הנחיית CSS מומלצת:

```css
.tour-cost-table {
  width: 370px;
  max-width: 370px;
  table-layout: fixed;
  margin-inline: auto;
}

.tour-cost-table th,
.tour-cost-table td {
  vertical-align: middle;
}

.tour-cost-table .tour-col-component {
  width: 125px;
  min-width: 125px;
  max-width: 125px;
  text-align: right;
}

.tour-cost-table .tour-col-quantity {
  width: 75px;
  min-width: 75px;
  max-width: 75px;
  text-align: center;
}

.tour-cost-table .tour-col-price,
.tour-cost-table .tour-col-total {
  width: 85px;
  min-width: 85px;
  max-width: 85px;
  text-align: center;
}
```

אם קיימים class names אחרים בפועל, לא חייבים להשתמש בדיוק בשמות האלה, אבל חובה שהתוצאה תהיה זהה:

* רכיב 125px ימין
* כמות 75px מרכז
* מחיר 85px מרכז
* סה״כ 85px מרכז
* טבלה כוללת קומפקטית יותר

דוגמה לתצוגה:

| רכיב         | כמות |    מחיר |    סה״כ |
| ------------ | ---: | ------: | ------: |
| יא 5         |   30 |   150 ₪ | 4,500 ₪ |
| מדריך מלווה  |    1 | 1,000 ₪ | 1,000 ₪ |
| הסעה-אוטובוס |    1 | 1,520 ₪ | 1,520 ₪ |
| סה״כ         |      |         | 7,020 ₪ |

חשוב:
לבטל את המבנה הישן שבו מדריך והסעה הם עמודות בתוך שורת הכיתה.
מעתה מדריך והסעה הם שורות עצמאיות בטבלה.

בדיקות חובה:

* יצירת הצעת מחיר מסוג סיור חדשה.
* הוספת שורת כיתה / קבוצה.
* הוספת שורת מדריך.
* הוספת שורת מיניבוס.
* הוספת שורת אוטובוס.
* בדיקה שכל שורה מחשבת כמות × מחיר.
* בדיקה שהסה״כ הכללי הוא סכום השורות.
* בדיקה שעמודת רכיב מיושרת לימין וברוחב 125px.
* בדיקה שעמודת כמות מיושרת לאמצע כולל כותרת וברוחב 75px.
* בדיקה שעמודת מחיר מיושרת לאמצע כולל כותרת וברוחב 85px.
* בדיקה שעמודת סה״כ מיושרת לאמצע כולל כותרת וברוחב 85px.
* בדיקה שהטבלה הכוללת קומפקטית יותר ולא רחבה מדי.
* בדיקה בתצוגה מקדימה.
* בדיקה ב־PDF / הדפסה.
* בדיקה שהצעות שאינן מסוג סיור לא נפגעו.

בסיום:

* לעדכן build.
* לעדכן Service Worker.
* לעדכן Cache version / cache busting.

דיווח בסיום:

* אילו קבצים שונו.
* האם בוצע שינוי רק לתבנית סיור.
* מה מבנה הטבלה החדש.
* האם הרוחבים והיישורים נבדקו בפועל.
* האם PDF נבדק בפועל.
* האם עודכן SW / Cache.
