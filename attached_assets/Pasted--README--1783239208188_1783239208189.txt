יש לקרוא קודם את README ואת מבנה הפרויקט, ואז לבצע תיקון ממוקד בלבד במסך הצעות מחיר / הסכמים.

הבעיה:
ב”הצעת מחיר קיץ” מתקבלת הודעה שלא נשמרו נתונים / אין תבנית, למרות שב-Supabase קיימת תבנית קיץ פעילה.

ממצאי בדיקה:

1. בטבלת `proposal_template_sections` קיימת תבנית פעילה:
   `template_key = summer`
2. בטבלת `proposal_activity_groups` קיימת קבוצה:
   `group_key = summer`
   `template_key = summer`
3. בקוד יש מיפוי של:
   `פעילויות קיץ`
   `קיץ תשפ״ו`
   אל `summer`
4. לכן הבעיה אינה חוסר תבנית במסד, אלא כנראה:

   * טעינת תבניות ריקה / ישנה במסך בגלל cache
   * או הצעות קיץ קיימות שאין להן `document_snapshot` / `document_html_snapshot`
   * או מצב שבו בעת שליחה/נעילה נדרש snapshot, אבל הוא לא נבנה מחדש מתוך התבנית הקיימת.

הקבצים המרכזיים לבדיקה:

* `frontend/src/screens/proposals-agreements.js`
* `frontend/src/api.js`
* כל קובץ שקשור ל-service worker / cache / versioning

הדרישה:
לתקן כך שבהצעת מחיר קיץ התבנית תיטען ותישמר תקין, בלי לפגוע בשנה הבאה, הצעה משולבת או סיור.

מה לבצע:

1. לוודא שבטעינת מסך הצעות מחיר הנתונים הבאים תמיד נטענים טריים מה-API/Supabase:

   * `proposalTemplateSections`
   * `proposalActivityGroups`
   * `proposalGroupAliases`
   * `proposalActivityPricing`

2. לוודא שפעילויות קיץ תמיד מזוהות כ־`summer`, גם אם הערך שמגיע מהרשומה הוא:

   * `summer`
   * `פעילויות קיץ`
   * `קיץ תשפ״ו`

3. לפני בניית תצוגה מקדימה / PDF / סימון כנשלח:
   אם להצעה אין `document_snapshot` או שאין לה `document_html_snapshot`,
   אבל `activity_type_group` מזוהה כ־`summer`,
   יש לבנות snapshot חדש מתוך `proposalTemplateSections` לפי `template_key = summer`.

4. לא להציג הודעה מטעה של “אין תבנית” אם קיימת תבנית ב־Supabase.
   במקום זה:

   * אם התבנית באמת לא נטענה למסך, להציג הודעה ברורה:
     “התבנית קיימת אך לא נטענה למסך. יש לרענן נתונים.”
   * ואם אין snapshot להצעה ישנה, לבנות אותו מחדש מהתבנית הפעילה.

5. לוודא שפעולת “סימון כנשלח” לא נכשלת בגלל snapshot חסר כאשר ניתן לבנות אותו מחדש מהתבנית הקיימת.

6. לא לשנות:

   * לוגיקת אישור וחתימה
   * הרשאות
   * מבנה טבלאות
   * חישובי מחירים
   * הצעות שנה הבאה
   * הצעות משולבות
   * סיור
   * PDF storage
   * שליחה / נעילה מעבר לתיקון הנדרש

7. להוסיף בדיקה ממוקדת:

   * הצעת קיץ עם `activity_type_group = summer`
   * קיימים `proposalTemplateSections` עם `template_key = summer`
   * גם אם אין `document_snapshot` קודם, המערכת מצליחה לבנות תצוגה מקדימה / snapshot ולא מחזירה “אין תבנית”.

8. בסיום:

   * לעדכן Service Worker
   * לעדכן Cache/version
   * לוודא שהדפדפן לא ממשיך להגיש קוד ישן

בסוף העבודה להחזיר לי:

1. מה הייתה הסיבה המדויקת לכשל
2. אילו קבצים שונו
3. מה נבדק
4. האם נדרש SQL — אם כן, לא להריץ בלי אישור, אלא להציג את ה-SQL בנפרד
