במערכת dashboard_system, במסך הצעות מחיר / proposals-agreements, נדרש תיקון מרוכז ומהיר לכמה בעיות בהצעה המודפסת ובצירוף הקטלוג.

מטרה:
לתקן את תצוגת ההצעה וה-PDF בצורה נקייה, בלי שינויי עומק מיותרים, בלי בדיקות ארוכות, ובלי לגעת בנתוני Supabase או בנוסחים שלא קשורים לתקלה.

קבצים עיקריים:
- frontend/src/screens/proposals-agreements.js
- frontend/src/styles/main.css
- service worker / cache version אם קיים

בעיה 1 — כפילות מחירים ושורות פעילות בהצעה
כרגע סעיף "הפעילות המוצעת" מציג גם תיאור וגם שורת פעילות דינמית עם גפ״ן / מפגשים / שעות / מחיר / כמות / סה״כ.
אחר כך אותו מידע חוזר שוב בטבלת "עלות ותנאי תשלום".

נדרש:
- בסעיף "הפעילות המוצעת" להציג רק את הטקסט התיאורי מתוך התבנית / Supabase.
- לא להוסיף שם proposalItemsListHtml או שורות מחיר דינמיות.
- שם הפעילות, כמות, מחיר יחידה וסה״כ יופיעו רק בטבלת "עלות ותנאי תשלום".
- אותו כלל גם להצעות קיץ, הצעות שנתיות והצעות משולבות.

בעיה 2 — מחיר שעתי לא מתעדכן אחרי שינוי מחיר
כאשר משנים ידנית מחיר יחידה, מחיר הקבוצה והסה״כ מתעדכנים, אבל המחיר לשעה נשאר מהמחירון המקורי.

דוגמה:
טכנולוגיות החלל – 16 שעות.
מחיר קבוצה שונה ל־7,500 ₪.
המערכת עדיין מציגה 500 ₪ לשעה, למרות שצריך להיות:
7,500 / 16 = 468.75 ₪ לשעה.

נדרש:
- כאשר משתנה input של מחיר יחידה / data-pa-item-price:
  - לחשב hourly_price מחדש לפי unit_price / hours_count.
  - רק אם hours_count קיים וגדול מ־0.
- לעדכן את hidden input בשם hourly_price.
- לעדכן גם את ה-info strip בשורה מיד לאחר שינוי מחיר.
- אם אין שעות או hours_count=0, לא להשאיר מחיר שעתי ישן מהמחירון.
- בתצוגה מקדימה ובהדפסה להשתמש במחיר השעתי המעודכן.

בעיה 3 — עמוד ריק עם פוטר בלבד
בהדפסה נוצר לפעמים עמוד ריק שמכיל רק את הפוטר/לוגו תחתון של תעשיידע.

נדרש:
- למנוע מצב שבו proposal-document-footer נשבר לעמוד חדש לבד.
- לא להשתמש ב-margin-top:auto בפוטר בזמן הדפסה אם זה גורם לשבירת עמוד.
- הפוטר צריך להיות מוצמד לתחתית עמוד ההצעה או להישאר בתוך גבולות ההצעה.
- אפשר להשתמש ב-position:absolute בתוך proposal-document, עם padding-bottom מתאים לגוף המסמך.
- לוודא שהפוטר לא מייצר עמוד ריק.

בעיה 4 — עמוד ריק לפני הקטלוג
לפני הקטלוג/נספח נוצר עמוד ריק בגלל div ריק עם class catalog-appendix-page-break.

נדרש:
- לא ליצור div ריק רק בשביל page break.
- להסיר את ההוספה של:
  catalog-appendix-page-break
  לפני הקטלוג.
- במקום זה להצמיד את שבירת העמוד לאלמנט הראשון האמיתי של הקטלוג/נספח.
- להוסיף class כמו pa-appendix-start רק לנספח הראשון בפועל.
- ב-CSS:
  .pa-appendix-start {
    break-before: page;
    page-break-before: always;
  }
- אם אין קטלוג אמיתי שנטען — לא ליצור page break בכלל.
- לוודא שאין עמוד ריק בין ההצעה לבין הקטלוג.

בעיה 5 — הדפסה לפני שהקטלוג נטען
כפתור "הדפסה / שמירה כ-PDF" פעיל גם אם הקטלוג/נספחים עדיין נטענים.

נדרש:
- כאשר include_catalog=true:
  - להשבית את כפתור ההדפסה בזמן טעינת הקטלוג.
  - להציג הודעה: "טוען נספחי קטלוג..."
  - להפעיל את הכפתור רק אחרי שכל הנספחים הרלוונטיים סיימו להיטען או נכשלו בצורה מבוקרת.
- אם PDF נספח חסר או לא נטען:
  - להציג הודעה קטנה למנהל בתצוגה המקדימה עם class no-print.
  - לא להסתפק רק ב-console.warn.
- לא להדפיס קטלוג חלקי בלי שהמשתמש יודע.

אופן ביצוע:
- תיקון ממוקד ומהיר בלבד.
- לא לשנות נוסחים עסקיים.
- לא לשנות מחירים במחירון.
- לא לשנות מבנה Supabase.
- לא להוסיף בדיקות ארוכות או כבדות.
- להוסיף לכל היותר בדיקות קצרות/ממוקדות אם כבר קיימת תשתית קלה.
- לאחר התיקון לעדכן service worker/cache version כדי שהדפדפן לא ישתמש בקוד ישן.

בדיקה ידנית קצרה לאחר התיקון:
1. הצעה רגילה בלי קטלוג — אין כפילות מחירים ואין עמוד ריק.
2. הצעה עם קטלוג — ההצעה מופיעה ואז הקטלוג מיד אחריה, בלי עמוד ריק באמצע.
3. שינוי מחיר יחידה מעדכן גם מחיר לשעה.
4. כפתור הדפסה לא פעיל בזמן טעינת קטלוג.
5. טבלת "עלות ותנאי תשלום" היא המקום היחיד שבו מופיעים מחירים וסה״כ.
