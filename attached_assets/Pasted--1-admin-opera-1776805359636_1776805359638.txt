עדכן את לוחות הבקרה לפי ההנחיות הבאות:

1. לוח הבקרה של אדמין ושל תפעול:
- עבור admin ועבור operations_reviewer, להציג לוח בקרה בסגנון האדמין.
- להעביר את אייקוני הניווט ל-header במרכז.
- לא להציג את בלוק "ניווט מהיר" בתוך גוף העמוד.
- לא להציג את בלוק "אבחון גיליונות" בתוך גוף העמוד.

2. אייקוני ניווט ב-header:
- להציג כאייקונים בלבד את היעדים:
  - dashboard
  - activities
  - finance
  - exceptions
  - permissions
- האייקונים יוצגו לפי ההרשאות בפועל של המשתמש.
- כלומר: להציג רק routes שמותרים למשתמש, ולא להציג אייקון ליעד שאין לו הרשאה אליו.
- להשתמש באותה לוגיקה של הרשאות/Routes הקיימת במערכת, בלי להקשיח ידנית הרשאות חדשות.

3. אדמין:
- admin יקבל את אייקוני ההידר לפי כל מה שמותר לו בפועל.

4. תפעול:
- operations_reviewer יקבל גם הוא את אייקוני ההידר, אבל רק לפי מה שמותר לו בפועל.
- אם אין לו הרשאה ליעד מסוים, לא להציג לו את האייקון שלו.
- המראה צריך להיות זהה לסגנון של האדמין, אבל התוכן לפי ההרשאות.

5. משתמשים רגילים:
- משתמשים שאינם admin ואינם operations_reviewer לא יקבלו את אייקוני הניווט האלה ב-header.
- לא להוסיף להם ניווט אדמיני.
- לא לשנות עבורם את מבנה ההידר מעבר למה שנדרש כדי להחריג רק admin ו-operations_reviewer.

6. כרטיסי KPI העליונים:
- הקטן את רוחב הכרטיסים.
- הצג רק את הכרטיסים הבאים:
  - תוכניות פעילות = קורסים + אפטר סקול, לפי החודש הנוכחי
  - פעילויות חד-יומיות = סדנאות + סיורים + חדרי בריחה + כל פעילות חד-יומית אחרת, לפי החודש הנוכחי
  - חריגות = רק קורסים
  - מדריכים פעילים = לפי כלל הפעילויות
  - סיומי קורסים = רק קורסים
- להסיר לחלוטין את הכרטיס "כספים פתוחים".

7. פילוח לפי מנהל פעילויות:
- עבור כל מנהל להציג רק:
  - תוכניות פעילות לחודש הנוכחי
  - מדריכים פעילים
  - חריגות
  - סיומי קורסים
- להסיר לחלוטין "כספים פתוחים" מהאזור הזה.
- "תוכניות פעילות" = קורסים + אפטר סקול
- "מדריכים פעילים" = לפי כלל הפעילויות של אותו מנהל
- "חריגות" = רק קורסים
- "סיומי קורסים" = רק קורסים

8. חשוב:
- לא להשאיר רכיבים ישנים לצד החדשים.
- לא להשאיר גם "ניווט מהיר" בגוף העמוד וגם אייקונים בהידר.
- לא להציג בנפרד קורסים / אפטר סקול / תוכניות אם הם כבר מאוחדים תחת "תוכניות פעילות".
- להשתמש בהרשאות וב-routes הקיימים במערכת כדי לקבוע אילו אייקונים יוצגו לאדמין ולתפעול.
