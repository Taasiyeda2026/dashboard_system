צריך לתקן את לוגיקת סטטוס בהצעות מחיר.

הבעיה:
הצעה בסטטוס "בוטל" יכולה להשתנות בטעות ל"מאושר".
זה לא תקין.

הדרישה:
סטטוס cancelled / בוטל הוא סטטוס סופי למחיקה בלבד.

כלומר:
1. הצעה בסטטוס cancelled / בוטל לא יכולה לעבור ל-approved / מאושר.
2. הצעה בסטטוס cancelled / בוטל לא יכולה לעבור ל-sent / נשלח.
3. הצעה בסטטוס cancelled / בוטל לא יכולה לעבור ל-pending_approval / ממתין לאישור.
4. הפעולות היחידות שמותרות להצעה שבוטלה:
   - צפייה
   - מחיקה
   - אפשרות עריכה רק אם קיימת היום לפי הרשאות, אבל לא שינוי לסטטוס מאושר
   - אפשרות שכפול להצעה חדשה אם קיימת מדיניות קיימת

התיקון ב־frontend/src/screens/proposals-agreements.js:

בפונקציה statusSelectHtml:
אם currentStatus הוא cancelled:
לא להציג select פעיל.
להציג badge בלבד:

if (currentStatus === 'cancelled') {
  return statusBadgeHtml(currentStatus);
}

בנוסף, בלוגיקת עדכון סטטוס:
לפני כל שינוי סטטוס לבדוק את הסטטוס הנוכחי.

אם currentActionRow.status מנורמל הוא cancelled:
להציג הודעה:
"הצעה שבוטלה נעולה. ניתן למחוק אותה או לשכפל להצעה חדשה."

ולעצור את הפעולה.

דוגמה:

const currentStatus = normalizeProposalStatus(currentActionRow?.status);

if (currentStatus === 'cancelled') {
  showToast('הצעה שבוטלה נעולה. ניתן למחוק אותה או לשכפל להצעה חדשה.', 'error');
  return;
}

גם ב־api.js:
בפונקציה updateProposalAgreementStatus להוסיף הגנה ברמת API:

לאחר קריאת currentRow:
const cs = normalizeProposalAgreementStatusForDb(currentRow.status);

if (cs === 'cancelled') {
  throw new Error('הצעה שבוטלה נעולה. ניתן למחוק אותה או לשכפל להצעה חדשה.');
}

החרגה:
אם בעתיד רוצים לאפשר שחזור מטעות, זה צריך להיות כפתור ייעודי "שחזור לטיוטה" ורק לאדמין.
כרגע לא להוסיף את זה.

בדיקות חובה:
1. הצעה בסטטוס בוטל מוצגת עם badge בלבד ולא select.
2. אי אפשר לשנות בוטל למאושר.
3. אי אפשר לשנות בוטל לנשלח.
4. כפתור מחיקה עדיין מופיע להצעה שבוטלה.
5. מחיקה של הצעה שבוטלה עובדת.
6. הצעות טיוטה עדיין ניתנות לעריכה / מחיקה.
7. הצעות ממתינות לאישור עדיין ניתנות לאישור לפי הרשאה.
8. לעדכן Service Worker ו־Cache בסיום.
