צריך לתקן את **תצוגה והדפסה של רשויות** באזור הפעילויות.

מדובר במסך רשויות וב־PDF/הדפסה של רשויות בלבד.
לא לשנות את לשונית “סידור עבודה” הכללית ולא טבלאות אחרות במערכת.

הדרישות:

1. שינוי כותרת ההדפסה

בהדפסה של רשויות, הכותרת העליונה כרגע היא:

“רשויות — סידור עבודה”

צריך לשנות ל:

“רשויות — פעילויות קיץ”

הכותרת צריכה להופיע גם בתצוגת ההדפסה וגם ב־PDF.

2. כל רשות מתחילה בדף חדש בהדפסה

בהדפסה/PDF, כל רשות צריכה להתחיל בעמוד חדש.

כלומר:

* רשות אשכול מתחילה בעמוד חדש
* הרשות הבאה מתחילה בעמוד חדש
* לא לערבב שתי רשויות באותו עמוד
* לא לשבור רשות חדשה באמצע עמוד קודם

CSS מומלץ:

```css
@media print {
  .authority-print-section {
    break-before: page;
    page-break-before: always;
  }

  .authority-print-section:first-child {
    break-before: auto;
    page-break-before: auto;
  }
}
```

יש להתאים את שם הקלאס לשמות הקיימים בקוד.

3. מבנה יום וטבלה

כרגע כותרות היום/תאריך מופיעות בצד ימין ורחוקות מהטבלה, ולכן זה נראה מנותק.

צריך שכל יום יהיה בלוק אחד:

כותרת יום/תאריך מעל הטבלה
ומיד מתחתיה הטבלה של אותו יום

לדוגמה:

יום ה׳ - 02/07/2026
[טבלה]

הכותרת של היום צריכה להיות מיושרת לימין של הטבלה ולא לקצה הימני של כל העמוד.

כלומר:

* הכותרת תהיה חלק מאותו בלוק של הטבלה
* הכותרת תהיה מעל הטבלה
* הכותרת תהיה באותו רוחב של הטבלה
* הכותרת והטבלה יהיו ממורכזות יחד

4. רוחב הטבלה בתצוגה ובהדפסה

גם בתצוגה במסך וגם בהדפסה/PDF, רוחב הטבלה הכולל צריך להיות קטן יותר:

רוחב טבלה כולל: 50%-55% מרוחב העמוד/אזור התוכן

עדיפות:
להתחיל מ־55%.
אם עדיין רחב מדי, לרדת ל־50%.

הטבלה צריכה להיות ממורכזת.

CSS מומלץ:

```css
.authority-day-block,
.authority-day-table-wrap {
  width: 55%;
  max-width: 55%;
  margin-inline: auto;
}

.authority-day-table {
  width: 100%;
  table-layout: fixed;
}

@media print {
  .authority-day-block,
  .authority-day-table-wrap {
    width: 55% !important;
    max-width: 55% !important;
    margin-inline: auto !important;
  }

  .authority-day-table {
    width: 100% !important;
    table-layout: fixed !important;
  }
}
```

אם 55% עדיין נראה רחב, לעבור ל־50%.

5. חלוקת עמודות בתוך הטבלה

בתוך הטבלה לשמור על חלוקה מאוזנת:

* שעות - צרה וברורה
* מדריך - בינונית
* כיתה - לא צרה מדי
* פעילות / סדנה - הרחבה ביותר, אבל לא מוגזמת

חלוקה מומלצת בתוך הטבלה:

```text
שעות: 20%
מדריך: 27%
כיתה: 20%
פעילות / סדנה: 33%
```

CSS לדוגמה:

```css
.authority-day-table .col-time {
  width: 20%;
}

.authority-day-table .col-instructor {
  width: 27%;
}

.authority-day-table .col-class {
  width: 20%;
}

.authority-day-table .col-activity {
  width: 33%;
}
```

אם אין classes לעמודות, להוסיף `colgroup` או class מתאים לכל עמודה.

6. למנוע כותרת יום לבד בסוף עמוד

בהדפסה אסור שכותרת של יום תישאר לבד בסוף עמוד והטבלה תעבור לעמוד הבא.

אם אין מקום לכותרת + תחילת הטבלה, להעביר את הכותרת והטבלה יחד לעמוד הבא.

CSS מומלץ:

```css
@media print {
  .authority-day-title {
    break-after: avoid-page;
    page-break-after: avoid;
  }

  .authority-day-title + table,
  .authority-day-title + .authority-day-table,
  .authority-day-title + .authority-day-table-wrap {
    break-before: avoid-page;
    page-break-before: avoid;
  }

  .authority-day-header-block {
    break-inside: avoid-page;
    page-break-inside: avoid;
  }

  .authority-day-table tr {
    break-inside: avoid-page;
    page-break-inside: avoid;
  }
}
```

חשוב:
לא למנוע שבירה של טבלה ארוכה אם יש הרבה שורות.
רק למנוע מצב שבו כותרת היום נפרדת מהטבלה.

7. הקו הכחול הארוך

הקו הכחול שמתפרס כמעט על כל העמוד נראה מיותר ומרחיק את הכותרת מהטבלאות.

צריך:

* או לקצר אותו לרוחב אזור התוכן בלבד
* או לוותר עליו
* בכל מקרה שלא יהיה קו ארוך על כל העמוד שמפריד בין הכותרת לטבלאות

8. כותרת רשות

בראש כל רשות להשאיר כותרת ברורה וקומפקטית:

שם הרשות
טווח תאריכים
מספר מסגרות / פעילויות

אבל לא לתפוס גובה גדול מדי.

לדוגמה:

אשכול
טווח תאריכים: 01/07/2026-31/08/2026
1 מסגרות · 36 פעילויות

9. ריווח בין ימים

להשאיר רווח קטן וברור בין יום ליום, אבל לא גדול מדי.

כל בלוק יום צריך להיראות כיחידה אחת:
כותרת יום + טבלה.

10. בדיקות חובה

בתצוגה במסך:

1. להיכנס ללשונית רשויות.
2. לפתוח רשות עם כמה ימי פעילות.
3. לוודא שכל יום מוצג כבלוק אחד.
4. לוודא שכותרת היום צמודה לטבלה.
5. לוודא שהטבלה ממורכזת.
6. לוודא שרוחב הטבלה הוא 50%-55%.
7. לוודא שאין כותרות יום שמופיעות רחוק מהטבלה.

בהדפסה/PDF:

1. לפתוח הדפסה של רשויות.
2. לוודא שהכותרת היא:
   “רשויות — פעילויות קיץ”
3. לוודא שכל רשות מתחילה בדף חדש.
4. לוודא שכותרת יום לא נשארת לבד בסוף עמוד.
5. לוודא שהטבלה ברוחב 50%-55% וממורכזת.
6. לוודא שאין חיתוך טקסט.
7. לוודא שאין גלישה מחוץ לעמוד.
8. לוודא שאין דפים ריקים מיותרים.
