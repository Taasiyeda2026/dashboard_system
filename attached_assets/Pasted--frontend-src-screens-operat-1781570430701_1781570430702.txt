תיקון נקודתי בלשונית **כמויות סדנאות** בעמוד **ניהול תפעול**.

קובץ:
`frontend/src/screens/operations-management.js`

בנוסף לעדכון SW:

* `frontend/sw.js`
* `sw.js`

לא SQL.
לא לשנות Supabase.
לא לשנות נתונים.
לא למחוק פעילויות.
רק תיקון תצוגה בטבלת המלאי / כמויות סדנאות.

## הדרישה

יש סדנאות שלא נדרש עבורן מלאי, ולכן לא צריך להציג אותן בטבלת **כמויות סדנאות / מלאי**.

להחריג מהטבלה:

* `תמיר`
* `עולם הביומימיקרי`
* `עולם הביומימיקרי הקסום`

אם השם מופיע כחלק משם ארוך יותר — גם להחריג.

## חשוב

ההחרגה היא רק מטבלת **כמויות סדנאות / מלאי**.

לא להסתיר את הפעילויות האלה:

* לא מסידור עבודה
* לא מלפי בתי ספר
* לא מספירת פעילויות כללית
* לא מהדפסה

רק לא להציג אותן בטבלת המלאי, כי לא נדרש להן מלאי.

## לוגיקה

להוסיף פונקציה קטנה, לדוגמה:

```js
function isWorkshopInventoryRequired(workshopName) {
  const name = String(workshopName || '').trim();
  if (!name) return false;

  const excluded = [
    'תמיר',
    'עולם הביומימיקרי',
    'עולם הביומימיקרי הקסום'
  ];

  return !excluded.some((term) => name.includes(term));
}
```

ואז בלשונית כמויות סדנאות, לפני בניית טבלת המלאי:

```js
const inventoryRows = rows.filter((row) =>
  isWorkshopInventoryRequired(getActivityName(row))
);
```

הטבלה תיבנה מתוך `inventoryRows` ולא מתוך כל `rows`.

## השפעה על כרטיסים / ספירות

אם יש ספירה של סדנאות בטבלת המלאי:
היא צריכה לספור רק סדנאות שנדרש להן מלאי.

לא להציג חוסר מלאי עבור סדנה שלא נדרש לה מלאי.

## בדיקות

1. לפתוח ניהול תפעול.
2. לעבור ללשונית כמויות סדנאות.
3. לוודא ש־`תמיר` לא מופיע בטבלת המלאי.
4. לוודא ש־`עולם הביומימיקרי` לא מופיע בטבלת המלאי.
5. לוודא ש־`עולם הביומימיקרי הקסום` לא מופיע בטבלת המלאי.
6. לוודא שהפעילויות האלה עדיין מופיעות בסידור עבודה ובלפי בתי ספר.
7. לוודא שאין עליהן חוסר מלאי.
8. להריץ:
   `npm run check:build`
9. לעדכן SW/cache.

## בסיום להחזיר

1. איזה קבצים שונו.
2. האם הסדנאות הוחרגו רק מטבלת המלאי.
3. האם הן עדיין מופיעות בפעילויות.
4. מה גרסת ה־SW החדשה.
5. האם build עבר.
