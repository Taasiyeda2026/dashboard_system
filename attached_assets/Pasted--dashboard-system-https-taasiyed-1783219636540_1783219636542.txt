יש לתקן בזהירות את תקלה/טעות המלאי במערכת dashboard_system.

קישורים:

* מערכת: https://taasiyeda2026.github.io/dashboard_system/
* Supabase: https://supabase.com/dashboard/org/dwyizsxtvfhmmofsecyl

לפני כל שינוי:

1. לקרוא את README / AGENTS / מסמכי ההנחיה בפרויקט.
2. לא לבצע שינוי עיוור.
3. לבדוק קודם את הקוד הקיים ואת מקור הנתונים בפועל.
4. לא לבצע שינויי SQL בשלב ראשון.
5. לא לשנות מבנה טבלאות.
6. לא למחוק נתונים.
7. לא לבצע ריפקטורינג רחב.
8. לא לשנות מסכים אחרים במערכת.

הבעיה לבדיקה:
במסך ניהול תפעול > ציוד ומלאי / מלאי פעילויות, נתוני המלאי מוצגים או מחושבים בצורה שגויה. יש לבדוק את מקור הטעות ולתקן רק את החלק שגורם להצגת מלאי לא נכונה.

חשוב מאוד:
מקור האמת של מלאי הסדנאות/פעילויות צריך להישאר בטבלת:
public.lists

עם השדות/הסינון הקיימים:

* category = activity_names
* type = workshop
* activity_type = workshop
* activity_no
* activity_name
* stock_quantity
* stock_group_key

אסור להחזיר את הלוגיקה למקורות ישנים כמו:

* category = workshop_stock
* type = inventory

אסור לבנות התאמה לפי שם פעילות בלבד אם קיימים activity_no או stock_group_key.
שם פעילות יכול להשתנות ולכן הוא לא צריך להיות המפתח המרכזי לחיבור מלאי.

מה צריך לבדוק בקוד:
חפש בקוד את כל המקומות שקשורים ל:

* ציוד ומלאי
* מלאי קיץ
* מלאי פעילויות
* workshop stock
* stock_group_key
* stock_quantity
* activity_names
* participants_count
* מלאי נדרש
* יתרה צפויה
* לא עודכן

דרישת תיקון:

1. לוודא שהמלאי נמשך מהמקור הנכון: public.lists לפי activity_names/workshop.
2. לוודא שהחיבור בין פעילות לבין מלאי נעשה לפי stock_group_key / activity_no ולא לפי שם תצוגה בלבד.
3. אם יש fallback ישן שמחזיר נתונים לא נכונים — לתקן אותו או להסיר אותו בזהירות.
4. אם participants_count חסר, ריק או לא תקין — להציג 0 ולא "לא עודכן".
5. לשמור על נוסחת יתרת המלאי הקיימת:
   יתרת מלאי = מלאי קיים - מלאי נדרש
6. לא לשנות את נתוני המלאי עצמם, אלא רק את שליפת הנתונים/החיבור/התצוגה אם שם מקור התקלה.
7. לא לפגוע בעריכת מלאי קיימת לפי stock_group_key מאחורי הקלעים.
8. לא להחזיר למסך שדות שכבר הוסתרו מהפרונט, כמו "סדנאות מקושרות" או "מלאי קיים", אם הם הוסתרו בכוונה.

בדיקות חובה אחרי התיקון:

1. לפתוח את ניהול תפעול > ציוד ומלאי.
2. לבדוק שהכמויות שמוצגות תואמות לנתונים ב־Supabase.
3. לבדוק פעילות שיש לה stock_group_key.
4. לבדוק פעילות שאין לה participants_count תקין ולוודא שמוצג 0.
5. לבדוק שלא נוצרה כפילות מלאי.
6. לבדוק שלא נמשך מלאי ממקור ישן.
7. לבדוק שלא נשברו מסכים אחרים בניהול תפעול.
8. אם בוצע שינוי בפרונט — לעדכן build, Service Worker ו־Cache version כדי שהשינוי ייטען בפועל באתר.

בסיום:
להחזיר לי סיכום קצר:

* מה הייתה סיבת התקלה
* באילו קבצים בוצע שינוי
* האם היה שינוי SQL או לא
* איך נבדק שהתיקון עובד
* האם עודכן Service Worker / Cache / Build
