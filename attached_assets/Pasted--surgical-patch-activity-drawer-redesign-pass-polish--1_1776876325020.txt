## Mandatory data mapping

### Activity groups

* `Programs` = `course` + `after_school`

  * main source: `data_long`
  * meetings source: `activity_meetings`

* `One-day activities` = `workshop` + `tour` + `escape_room`

  * main source: `data_short`
  * date source: same `data_short`
  * no `activity_meetings` source for these

### Private note source

* operational private note source = `operations_private_notes`

### Day derivation

Never treat day as editable or as standalone source-of-truth.
Always derive day from actual stored date.

## 4. 📚 section

### View mode

Render only meaningful summary values:

* funding
* class (combined from grade + class_group)
* hours
* day

Do **not** render again:

* activity name
* school
* authority
* grade as separate field
* class_group as separate field
* price

Do not render useless placeholders that make the block look empty.
If a field is missing, prefer not rendering a dead summary item over rendering large empty UI.

### Edit mode

Must render for **all types**, not only for programs:

* type-specific activity selector
* funding
* price
* school
* authority
* grade
* class_group
* start_time
* end_time

### Type-specific activity label

Use type-specific label in edit mode for all activity types:

* course -> `שם קורס`
* after_school -> `שם חוג אפטרסקול`
* workshop -> `שם סדנה`
* tour -> `שם סיור`
* escape_room -> `שם פעילות`

Important:
Current logic wrongly limits this to `isProgram`.
Fix it so non-program types also get correct selector + correct label.

### Authority input

`authority` in edit mode must be free text input, not hardcoded dropdown.

### Hours computation

Do not use `row.hours` as display source-of-truth.
Compute hours from:

* `start_time`
* `end_time`

Display format example:
`11:20-12:05`

### Day computation

Do not use `row.day` as display source-of-truth.
Compute day from actual date:

* Programs: first meeting date from `activity_meetings`
* One-day activities: single date from `data_short`

Display the summary day in short Hebrew weekday form:

* א
* ב
* ג
* ד
* ה
* ו
* ש

## 5. 📅 section

This section must behave like a strict state machine.

### View mode

Render only:

* progress row
* end-date row
* compact meeting chips grid
* one `+עוד / פחות` control only if needed

Do not render in view mode:

* edit cards
* date inputs
* chain/single toggle
* save/cancel buttons

### Edit mode

Render only:

* end-date row (read-only)
* editable meeting cards grid
* date picker for each meeting
* weekday short abbreviation next to each meeting
* chain/single toggle only when allowed
* add-meeting button only when allowed
* save/cancel controls

Do not render in edit mode:

* view chips
* duplicate `+עוד`
* mixed view/edit structures together

### End date rule

End date is always derived from the last meeting date.
Never directly editable.

### Layout rule

* Programs:

  * view = 3 columns
  * edit = 2 columns

* One-day activities:

  * exactly one date only
  * no add button
  * no chain/single toggle

### Weekday rendering per meeting

For every displayed meeting/date, render short weekday abbreviation next to the date:

* א / ב / ג / ד / ה / ו / ש

Current implementation problem to fix:

* view currently renders date only
* edit currently renders full weekday string
  Replace both with short weekday abbreviation.
