בדוק ותקן את בורר הצבע במערכת dashboard_system.

ריפו:
https://github.com/Taasiyeda2026/dashboard_system.git

המטרה:
בורר הצבע חייב לשנות את צבע המערכת מיד בלחיצה, לשמור את הבחירה מקומית, לטעון אותה אחרי רענון, ואם קיימת תמיכה בשרת — לשמור גם בשרת. אסור שהבורר רק יוצג בלי להשפיע.

קבצים מרכזיים לבדיקה:
frontend/src/main.js
frontend/src/styles/main.css
frontend/src/api.js
backend/actions.gs
backend/router.gs
backend/sheets.gs
sw.js
index.html

הבעיה:
ב־frontend/src/main.js כבר קיימות פונקציות הקשורות לבורר הצבע:
ACCENT_COLORS
normalizeAccentName
accentNameFromStorage
applyGlobalAccent
bindAccentPickerOnce

אבל בפועל בורר הצבע לא משנה את צבע המערכת. יש לוודא שכל השרשרת מחוברת:
1. HTML של הבורר קיים בפועל בתוך ה־shell.
2. הכפתור כולל data-accent-picker-btn.
3. הפופאפ כולל data-accent-picker-popover.
4. כל צבע כולל data-accent-swatch ו־data-accent תקין.
5. bindAccentPickerOnce() נקראת אחרי שה־shell נוצר.
6. applyGlobalAccent() מופעלת גם בעליית האפליקציה וגם אחרי התחברות/טעינת clientSettings.
7. משתני CSS מתעדכנים על document.documentElement.
8. אין CSS שמדרס את הצבעים עם ערכים קשיחים במקום var(--ds-accent).
9. אין service worker/cache שמגישים קובץ main.js ישן.

יש לבצע תיקון מלא:

א. ב־frontend/src/main.js:
לאחר כל יצירת shell, בתוך bindShell() או מיד אחרי app.innerHTML = shell(...), לקרוא:

applyGlobalAccent();

ה. לתקן את applyGlobalAccent כך שתעדכן גם את כפתור הבורר וגם את כל הסוואצ׳ים:

function applyGlobalAccent(name = accentNameFromStorage()) {
  const selected = normalizeAccentName(name) || 'blue';
  const colors = ACCENT_COLORS[selected] || ACCENT_COLORS.blue;
  const root = document.documentElement;

  root.style.setProperty('--ds-accent', colors.accent);
  root.style.setProperty('--ds-accent-hover', colors.hover);
  root.style.setProperty('--ds-accent-soft', colors.soft);
  root.style.setProperty('--ds-interactive-selected', colors.soft);
  root.style.setProperty('--ds-activities-stripe', colors.stripe);
  root.style.setProperty('--ds-activities-stripe-hover', colors.stripeHover);
  root.style.setProperty('--ds-focus-ring', `0 0 0 2px color-mix(in srgb, ${colors.accent} 24%, transparent)`);
  root.style.setProperty('--ds-focus-ring-strong', `0 0 0 3px color-mix(in srgb, ${colors.accent} 30%, transparent)`);

  root.dataset.dsAccent = selected;

  document.querySelectorAll('[data-accent-picker-btn]').forEach((btn) => {
    btn.style.backgroundColor = colors.accent;
    btn.dataset.currentAccent = selected;
  });

  document.querySelectorAll('[data-accent-swatch]').forEach((swatch) => {
    swatch.classList.toggle('is-active', swatch.dataset.accent === selected);
  });

  return selected;
}

ו. לתקן את אירוע הלחיצה כך שלא יישבר אם api.saveClientSetting לא קיים:

const saveAccentToServer = async (selected) => {
  if (typeof api.saveClientSetting !== 'function') return;
  try {
    await api.saveClientSetting({ key: 'accent_color', value: selected });
  } catch (err) {
    console.warn('[accent-picker] Server save failed; local choice remains active:', err);
  }
};

ז. ב־frontend/src/api.js:
לבדוק אם קיימת פונקציה saveClientSetting.
אם לא קיימת, להוסיף אותה לאובייקט api ולחבר אותה לפעולת backend בשם saveClientSetting.

דוגמה:
saveClientSetting: ({ key, value }) => request('saveClientSetting', { key, value })

וכן להוסיף את saveClientSetting ל־MUTATING_ACTIONS כדי שמערכת ה־API תתייחס אליה כפעולת כתיבה.

ח. ב־backend/router.gs / actions.gs:
אם אין handler בשם saveClientSetting, להוסיף handler ששומר את ההגדרה בטבלת/גיליון ההגדרות הקיים.
הפעולה צריכה לקבל:
key
value

ולאפשר לפחות:
accent_color
theme_accent

אם אין טבלת הגדרות מסודרת, לא לשבור את המערכת. במקרה כזה השמירה המקומית ב־localStorage מספיקה, אבל לא להשאיר קריאת API שבורה.

ט. ב־buildClientSettingsFromLists או בכל מקום שבו נטענות הגדרות לקוח:
לוודא ש־accent_color או theme_accent חוזרים לתוך state.clientSettings.
אם יש accent_color בשרת, הוא צריך להשפיע על applyGlobalAccent אחרי bootstrap.

י. בעיית cache / service worker:
לבדוק אם sw.js או service worker ישן מגישים main.js ישן.
יש לוודא:
1. להעלות CACHE_VERSION ב־sw.js.
2. אם SERVICE_WORKER_ENABLED=false, לבצע unregister לכל service worker קיים בזמן טעינת האפליקציה.
3. לנקות caches קיימים:
   caches.keys().then(keys => Promise.all(keys.map(key => caches.delete(key))));
4. לא להשאיר מצב שבו הדפדפן טוען main.js ישן ללא קוד בורר הצבע.

יא. בדיקת קבלה:
לאחר התיקון:
1. נכנסים למערכת.
2. לוחצים על בורר הצבע.
3. בוחרים ירוק.
4. הצבע של הסיידבר, כפתורים פעילים, כותרות/דגשים שמשתמשים ב־var(--ds-accent) משתנה מיד.
5. מרעננים את הדף.
6. הצבע הירוק נשאר.
7. מחליפים לסגול.
8. הצבע משתנה מיד.
9. אין שגיאה בקונסול.
10. אם api.saveClientSetting לא קיים או נכשל, הבחירה עדיין נשמרת מקומית ולא נשברת.

לא לבצע שינוי עיצובי רחב מעבר לתיקון בורר הצבע.
לא לשנות שמות מסכים.
לא לגעת במבנה נתונים של פעילויות.
לא לשנות הרשאות.
לתקן רק את השרשרת: HTML של הבורר → event binding → applyGlobalAccent → CSS variables → localStorage → server save אם קיים → cache/service worker.
