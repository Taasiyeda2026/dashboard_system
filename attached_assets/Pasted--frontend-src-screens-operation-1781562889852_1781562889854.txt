תיקון נקודתי להדפסת **סידור עבודה** בעמוד **ניהול תפעול**.

קובץ:
`frontend/src/screens/operations-management.js`

אם יש CSS הדפסה בקובץ אחר — לעדכן שם נקודתית.

לא SQL.
לא לשנות Supabase.
לא לשנות נתונים.
לא לשנות את המסך הרגיל אם לא חייבים.
התיקון הוא רק לתצוגת ההדפסה.

בנוסף לעדכן Service Worker/cache בסיום אם הקובץ השתנה:

* `frontend/sw.js`
* `sw.js`

---

## הבעיה בהדפסה

ההדפסה כרגע רחבה ואוורירית מדי.

יש יותר מדי רווחים, הטבלה גדולה מדי, ופרטי בית הספר מוצגים בכמה שורות במקום בשורה אחת.

צריך להכניס יותר מידע בכל עמוד A4.

---

## מבנה הדפסה רצוי

כל בלוק של יום + בית ספר צריך להיות קומפקטי.

### כותרת בלוק

במקום להציג את פרטי בית הספר בכמה שורות, להציג בשורה אחת:

`יום ג׳ · 07/07/2026 | רשות: רחובות | בית ספר: בן גוריון | כתובת: ... | איש קשר: — | טלפון: —`

כלומר:

* יום ותאריך
* רשות
* בית ספר / מסגרת
* כתובת
* איש קשר
* טלפון

כולם באותה שורה או בשתי שורות קצרות לכל היותר.

לא לפתוח כרטיס גדול עם הרבה רווחים.

---

## טבלת פעילויות בתוך הבלוק

הטבלה צריכה להיות צרה וקומפקטית.

עמודות:

1. שעה
2. פעילות
3. שכבה / כיתה

אם מדפיסים “כל המדריכים”, להוסיף גם:
4. מדריך

אם מדפיסים מדריך יחיד — לא להציג עמודת מדריך.

---

## רוחב הטבלה

טבלת הפעילויות בתוך כל בלוק לא צריכה להיות ברוחב מלא אם אין צורך.

להגדיר:

* רוחב טבלה: `65%` עד `75%`
* יישור לימין
* לא למרכז
* לא לפתוח רוחב מלא

CSS מוצע:

```css
.ops-print-activities-table {
  width: 70%;
  margin-inline-start: auto;
  margin-inline-end: 0;
  border-collapse: collapse;
}
```

אם הטבלה עדיין רחבה מדי:
להשתמש ב־`60%`.

## צמצום רווחים בהדפסה

להקטין משמעותית:

* padding של בלוק
* margin בין בלוקים
* גובה שורות
* גודל כותרות
* רווחים מעל ומתחת לטבלה

```css
  border: 1px solid #cfd8dc;
  page-break-inside: avoid;

}

.ops-print-place-meta {

}

.ops-print-activities-table th,
.ops-print-activities-table td {
  padding: 3px 5px;
  font-size: 10.5px;
  line-height: 1.2;
}
```

## לא לחזור על מידע מיותר

בתוך שורות הטבלה לא להציג שוב:

* רשות
* בית ספר
* כתובת
* איש קשר
* טלפון

המידע הזה יוצג רק פעם אחת בכותרת הבלוק.

בתוך הטבלה רק:

* שעה
* פעילות
* שכבה / כיתה
* מדריך רק אם מדפיסים כל המדריכים

---

## שבירת עמודים

כל בלוק יום + בית ספר צריך להשתדל לא להיחתך באמצע:

```css
.ops-print-place-block {
  break-inside: avoid;
  page-break-inside: avoid;
}
```

אבל אם יש הרבה פעילויות, עדיף לא ליצור בלוקים ענקיים עם רווחים.

## מטרה כמותית

צריך להגיע למצב שבעמוד A4 אחד נכנסים כמה בלוקים, לא רק 2–3.

המטרה:

* לפחות 5–7 בלוקים קצרים בעמוד אם יש מעט פעילויות בכל מקום.
* טבלאות קצרות וצפופות.
* בלי כרטיסים גדולים.

## בדיקות

1. לבחור מדריך.
2. ללחוץ `הדפס סידור עבודה`.
3. לוודא שפרטי בית הספר מופיעים בשורה אחת / שורה וחצי.
4. לוודא שהטבלה בתוך הבלוק צרה ולא ברוחב מלא.
5. לוודא ששורות הטבלה נמוכות וקומפקטיות.
6. לוודא שלא חוזרים בכל שורה על בית ספר / כתובת / איש קשר.
7. לוודא שנכנסים יותר בלוקים בעמוד.
8. לבדוק PDF preview.
9. להריץ `npm run check:build`.
10. לעדכן SW/cache אם צריך.

---

## בסיום להחזיר

1. איזה קובץ שונה.
2. האם פרטי בית הספר עברו לשורה קומפקטית.
3. האם טבלת הפעילויות צומצמה ברוחב.
4. האם נכנס יותר מידע בעמוד.
5. האם עודכן SW/cache.
6. האם build עבר.
