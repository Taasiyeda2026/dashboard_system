# מקור נתונים למסך Dashboard — גיליון `dashboard`

## עקרון

מסך **Dashboard** בפרונט נטען דרך הפעולה `dashboardSheet` בלבד (`api.dashboardSheet({ month })`).  
אין במסלול הרגיל שימוש ב־`dashboardSnapshot`, ב־`actionDashboard_`, בסריקת `data_short` / `data_long`, או ב־read model כמקור לתצוגת הדשבורד.

## גיליון וטווח קריאה (קריאה אחת)

- שם הגיליון: **`dashboard`** (`DASHBOARD_SHEET_NAME_`).
- **טווח מאוחד אחד:** `getRange(1, 1, 17, 23)` — כלומר **A1:W17** (איחוד שלושת הטבלאות המפורטות להלן).
- אין קריאות נוספות לגיליונות אחרים.

## מבנה כללי (בכל בלוק)

- **שורה 1** — כותרת מנהל / `all`.
- **שורה 2** — שם אזור / מחוז.
- **`label_col`** — שמות מדדים (טקסט).
- **`month_cols`** — ערכים לפי חודש; **כותרות חודש בשורה 2** באותן עמודות (מזוהות כ־`YYYY-MM` או תאריך).
- **שורות 3–8** — ספירת פעילויות לפי סוג.
- **שורה 10** — תאריכי סיום (`end_date`).
- **שורות 12–14** — חריגות (איחור סיום, פערים תפעוליים, סה״כ חריגות).
- **שורה 16** — מספר מדריכים ייחודיים.
- **שורה 17** — שמות מדריכים (מופרדים בפסיק/נקודה-פסיק), ללא כפילות בטקסט.

**לא** משתמשים במבנה «עמודה A = אזור, B = מדד» לכל הגיליון — אלא **שלושה טווחי טבלה קבועים** עם `label_col` + `month_cols` לכל אזור.

## זיהוי חודש מבוקש

בבלוק **כל הארץ**, בשורה **2**, בעמודות **B–G** (ובמקביל בצפון/דרום ב־`month_cols`), מחפשים עמודה שהתא בשורה 2 שלה מתאים ל־`month` מהבקשה (`YYYY-MM`).  
אם לא נמצא — `DASHBOARD_MONTH_NOT_FOUND:<ym>` (בפרונט מתורגם לעברית).

## מיפוי ל־payload (תמצית)

- **סה״כ חד־יומי / KPI short:** סכום שורות 5–8 (`after_school` + `workshop` + `tour` + `escape_room`) בבלוק כל הארץ לעמודת החודש.
- **סה״כ תוכניות / KPI long:** שורה 4 (`course`).
- **סיומי קורסים:** שורה 10 (`end_date`).
- **חריגות:** שורה 14 (`course_total_exceptions`).
- **מדריכים:** שורה 16 (מספר) ו־17 (שמות); אם 16 ריק — נספרים שמות מ־17.

השדה **`_is_dashboard_sheet: true`** מסמן תגובה ממקור הגיליון.

## למה לא snapshot / read model לדשבורד

קריאה ישירה לגיליון המוכן מפחיתה עומס ומונעת תלות ב־read model לטעינת המסך.

## מה אסור במסלול Dashboard הרגיל

- `actionDashboard_`
- `dashboardSnapshot` כמקור למסך
- `data_short` / `data_long` / `activity_meetings` לצורך מסך זה
