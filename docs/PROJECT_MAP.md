# Project Map

Short map for keeping Codex/Cursor work scoped. Do not delete or move files based on this map alone.

## Live system code

- `frontend/src/` ‚Äî main application source.
- `frontend/src/main.js` ‚Äî app bootstrap, routing, shared screen loading.
- `frontend/src/api.js` and `frontend/src/supabase-client.js` ‚Äî API/data access clients.
- `frontend/src/state.js`, `frontend/src/config.js`, `frontend/src/permissions.js` ‚Äî shared state, configuration, and permission helpers.
- `supabase/migrations/` ‚Äî database migrations for the live Supabase-backed system.
- `scripts/` ‚Äî targeted maintenance, build, migration, sync, and verification scripts.
- Root `index.html`, `vite.config.js`, `package.json` ‚Äî app entry/build configuration.

## Screens

- `frontend/src/screens/` ‚Äî active screen modules, for example dashboard, week/month, activities, operations, finance, catalog, certificates, invitations, permissions, and proposals/agreements.
- `frontend/src/screens/shared/` ‚Äî shared screen-level helpers/components.
- `frontend/src/styles/` ‚Äî frontend styling used by screens.

## Service Worker / PWA

- `frontend/sw.js` ‚Äî source Service Worker.
- `sw.js` ‚Äî root Service Worker copy used by the deployed app.
- `frontend/assets/pwa/` ‚Äî source PWA icons/assets.
- `dist/sw.js` and `dist/manifest.json` ‚Äî generated build output; inspect only for build/deploy verification.

## Tests

- `tests/` ‚Äî focused and legacy Node test files.
- Prefer a specific relevant test, for example `node --test tests/proposals-agreements-screen.test.mjs`.
- Do not run `npm run test:all:legacy` unless explicitly requested.

## Folders to avoid in normal tasks

- `attached_assets/` ‚Äî pasted user files, screenshots, images, and imported artifacts. Do not scan unless the task explicitly asks for them.
- `dist/` ‚Äî generated build output. Do not scan unless the task is about build/deploy output.
- `frontend/public/catalog/` ‚Äî large public catalog PDFs/data/static pages. Read only for catalog-specific tasks.
- `docs/prompts/` ‚Äî archived prompt/task text. Read only when prompt history is explicitly needed.
- `artifacts/` and `prototypes/` ‚Äî mockups, experiments, and prototypes, not normal live app context.
- `node_modules/` ‚Äî dependencies; never scan for project understanding.

## Cache / precache exclusions

Do not add these folders wholesale to Service Worker cache or precache lists:

- `attached_assets/`
- `dist/`
- `docs/`
- `docs/prompts/`
- `tests/`
- `artifacts/`
- `prototypes/`
- `frontend/public/catalog/appendices/`
- `frontend/public/catalog/summercatalog/` unless a catalog task explicitly requires selected files.
- `node_modules/`

Cache only app-shell files and small, necessary runtime assets. Large PDFs, pasted files, generated output, and test/archive folders should stay out of precache.
